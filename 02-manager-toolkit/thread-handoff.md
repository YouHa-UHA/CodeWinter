# Thread Handoff 模板

适用于当前线程需要另一个线程继续接力时。

```text
请只基于已确认信息输出标准 handoff，不要猜测。

请包含：
1. 背景与目标
2. 当前已确认状态
3. 已完成工作
4. 涉及的文件、模块、服务或应用
5. 关键 API、配置、状态口径、SQL 语义
6. 约束与失败风险
7. 推荐接力角色或接力能力画像
8. 下一个线程第一步先做什么
9. 下一个线程需要完成的动作
10. 最小成功信号
11. 不建议对方先做什么
12. 推荐验证方式
13. 最危险的误判点
14. 仍待确认的信息

如果这是一次正式协作请求，请同步补充：
1. 当前线程状态卡
2. 对应的协作请求卡
```
